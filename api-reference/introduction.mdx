# API Overview

## Base URL

```text
https://your-deployment-url/api
```

For local development the default is:

```text
http://localhost:4000/api
```

## Auth Model In The Current Code

<Warning>
The checked-in backend does not currently enforce bearer-token auth on most routes. The CLI sends `Authorization: Bearer ...`, but self-hosters should treat the API as trusted-boundary infrastructure until auth middleware is added.
</Warning>

Authentication-related behavior that does exist today:

- `POST /users/register` stores the GitHub token you provide during onboarding.
- `POST /github/webhooks` expects a GitHub token in the request body so it can create repository webhooks with Octokit.
- `POST /webhooks/github` verifies `x-hub-signature-256` only when `GITHUB_WEBHOOK_SECRET` is configured.

## Endpoint Groups

### Users

| Method | Endpoint | Description |
|--------|----------|-------------|
| `POST` | `/users/register` | Register a new user |
| `GET` | `/users/:userId/stats` | Get full user stats |
| `POST` | `/users/:userId/xp` | Award XP |
| `GET` | `/users/:userId/activity` | XP activity log |
| `GET` | `/users/:userId/xp` | XP history alias |
| `GET` | `/users/:userId/loot` | User inventory |
| `POST` | `/users/:userId/prestige` | Prestige reset |
| `POST` | `/users/:userId/achievements/:achievementId/share` | Share URL generation |

### Quests

| Method | Endpoint | Description |
|--------|----------|-------------|
| `GET` | `/users/:userId/quests/daily` | List daily quests |
| `GET` | `/users/:userId/quests/weekly` | List weekly boss quests |
| `POST` | `/users/:userId/quests/:questId/complete` | Complete a quest |

### Leaderboard

| Method | Endpoint | Description |
|--------|----------|-------------|
| `GET` | `/leaderboard/global` | Global leaderboard |
| `GET` | `/users/:userId/leaderboard/friends` | Friends leaderboard |

### Social

| Method | Endpoint | Description |
|--------|----------|-------------|
| `GET` | `/users/:userId/friends` | List friends |
| `POST` | `/users/:userId/friends` | Add friend |
| `DELETE` | `/users/:userId/friends/:friendId` | Remove friend |

### Crafting & Guilds

| Method | Endpoint | Description |
|--------|----------|-------------|
| `GET` | `/crafting/recipes` | List craftable recipes |
| `POST` | `/crafting/:userId/craft/:recipeCode` | Craft a recipe |
| `POST` | `/guilds/create` | Create a guild |
| `POST` | `/guilds/join` | Join a guild |
| `POST` | `/guilds/leave` | Leave guild |
| `POST` | `/guilds/invite` | Invite user |
| `GET` | `/guilds/user/:userId` | User's guild |
| `GET` | `/guilds/:guildId` | Guild details |
| `GET` | `/guilds/:guildId/quests` | Guild quests |

### Notifications

| Method | Endpoint | Description |
|--------|----------|-------------|
| `GET` | `/notifications/:userId/integrations` | List saved integrations |
| `POST` | `/notifications/:userId/integrations` | Save Slack or Discord webhook |
| `DELETE` | `/notifications/:userId/integrations/:type` | Remove integration |

### Webhooks

| Method | Endpoint | Description |
|--------|----------|-------------|
| `POST` | `/webhooks/github` | GitHub webhook receiver |
| `POST` | `/github/webhooks` | Auto-configure repo webhook |
| `GET` | `/health` | Health check |

## WebSocket Events

Connect to the server root, not the `/api` prefix:

```text
https://your-deployment-url
```

The backend emits:

- `leaderboard:update`
- `loot`
- `craft`
- `battle:update`
- `battle:win`

See the [Realtime API](/api-reference/realtime) page for event payload examples.
